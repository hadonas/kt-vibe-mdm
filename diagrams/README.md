# MDM 시스템 다이어그램

이 폴더는 MDM (Monolithic Document Management) 시스템의 아키텍처 및 시퀀스 다이어그램을 포함합니다.

## 📁 파일 구조

### 아키텍처 다이어그램
- **`architecture.md`** - 시스템 전체 아키텍처 및 기술 스택

### 시퀀스 다이어그램
- **`document-registration-sequence.md`** - 문서 등록 및 승인 프로세스
- **`rag-search-sequence.md`** - RAG 검색 및 시스템 관리 프로세스
- **`file-download-sequence.md`** - 파일 다운로드 프로세스

## 🎯 다이어그램 개요

### 1. 아키텍처 다이어그램
- 전체 시스템 구조
- 서비스별 계층 구조
- 데이터 흐름 및 의존성
- 기술 스택 정보

### 2. 문서 등록 시퀀스
- **일반 유저**: 파일 업로드 → 문서 정보 입력 → 등록 요청 생성
- **관리자 유저**: 승인 요청 조회 → 상세 검토 → 승인/반려 결정

### 3. RAG 검색 시퀀스
- **일반 유저**: 쿼리 입력 → 벡터 검색 → AI 답변 생성 → 결과 표시
- **관리자 유저**: 시스템 관리 → 벡터 인덱스 재구성

### 4. 파일 다운로드 시퀀스
- **일반 유저**: 파일 다운로드 요청 → 파일명 추출 → 다운로드 실행
- **관리자 유저**: 파일 관리 → 계층 구조 조회 → 카테고리별 파일 조회

## 🔧 사용 방법

각 다이어그램은 Mermaid 형식으로 작성되어 있으며, 다음 도구에서 렌더링할 수 있습니다:

## 📊 다이어그램 특징

### 시퀀스 다이어그램
- **참여자**: 일반 유저, 관리자 유저, 프론트엔드, 백엔드 서비스, 데이터베이스
- **서비스 분리**: File Service, Ingest Service, Approval Service, Chat Service, Vector Search Service, Public Service
- **데이터 흐름**: 요청/응답 패턴, 에러 처리, 비동기 처리

### 아키텍처 다이어그램
- **계층 구조**: User Layer, Frontend Layer, API Gateway Layer, Service Layer, Data Layer
- **외부 서비스**: AI Service (Ollama/OpenAI)
- **데이터 저장소**: MongoDB, GridFS, FAISS Vector Store
